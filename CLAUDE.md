# MiniCMS Pro — Claude Code Reference

## Project Overview

**MiniCMS Pro v10** is a modular, community-focused PHP Content Management System. It features a forum, member directory, messaging, downloads, news, and shoutbox — all built on an extensible plugin ("infusion") system.

- **Dev server**: XAMPP at `http://localhost/MiniCMS`
- **Language**: Lithuanian UI defaults (i18n built-in)
- **Status**: Active development

---

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Backend | PHP 8.0+, procedural + OOP hybrid |
| Database | MySQL/MariaDB via PDO (no ORM) |
| Frontend | Bootstrap 5, jQuery, FontAwesome 7 |
| Email | PHPMailer v6.12.0 (Composer) |
| Icons | FA7 at `/includes/plugins/webicon/fa7/` |
| Drag & Drop | SortableJS (panel management) |

## Architecture Patterns

### Hybrid Procedural + OOP

- **Procedural layer**: Functions in `/includes/*.php` and `/includes/functions/` — used in page files directly
- **OOP layer**: Classes in `/includes/classes/MiniCMS/` — for complex services (Auth, Mail, Installer, InfusionSdk)
- Page files (e.g., `forum.php`, `profile.php`) are procedural scripts, not controllers

### Global Database Access

```php
// Available everywhere after bootstrap.php
$pdo = $GLOBALS['pdo']; // ProfiledPDO instance (extends PDO)
```

Always use **prepared statements** — no string interpolation in SQL.

### Bootstrap Flow

Every page includes at the top:
```php
require_once 'maincore.php';   // defines BASEDIR, SITE_URL, path constants
require_once 'config.php';     // DB credentials, app constants
require_once ROOT_PATH . 'includes/bootstrap.php';  // session, headers, DB, autoloader
```

## Database

- **Driver**: PDO with MySQL/MariaDB
- **No ORM** — raw SQL with prepared statements throughout
- **Profiling**: `ProfiledPDO` wraps PDO to track query counts/times (dev mode)

### Core Tables

| Table | Purpose |
|-------|---------|
| `users` | User accounts (id, username, email, password, role_id, is_active, status, avatar) |
| `roles` | Roles (super_admin, admin, moderator, member, guest) with `level` |
| `permissions` | Permission definitions (slug-based) |
| `role_permissions` | Role ↔ permission mapping |
| `settings` | Key-value site configuration (modules use `<module>_` prefix keys) |
| `site_settings` | Legacy fallback table older installs may still have |
| `navigation_links` | Hierarchical nav menu (parent_id for nesting) |
| `infusions` | Installed module registry |
| `infusion_panels` | Panel widgets with position/sort_order |
| `infusion_versions` | Module version tracking for upgrades |
| `audit_logs` | Admin action audit trail |
| `auth_rate_limits` | Login attempt tracking for rate limiting |
| `password_resets` | Password reset tokens |
| `moderation_logs` | Moderation action history |

### Migrations

Schema versions tracked in `/database/*.sql` (currently 1–13). New schema changes go in a new numbered `.sql` file and should be reflected in `database.sql`.

## Authentication & Authorization

### Session Auth

- `includes/auth.php` — login/logout logic
- `includes/classes/MiniCMS/Auth/AuthService.php` — OOP auth service
- Sessions: HttpOnly, Secure, SameSite=Lax, strict mode
- Rate limiting: 5 failures per email → 15-min lockout; 20 per IP

### Checking Permissions

```php
// Check single permission
if (!has_permission($pdo, $user_id, 'users.edit')) {
    http_error(403);
}

// Common permission slugs:
// admin.access, settings.manage, users.view, users.edit, users.delete,
// users.status, roles.manage, permissions.manage, infusions.manage,
// themes.manage, forum.view, forum.admin, news.view, news.admin,
// downloads.view, downloads.upload, downloads.admin, shoutbox.moderate
```

### Admin Guard

Every admin page starts with:
```php
require_once '_guard.php';  // checks admin.access permission, redirects if not
```

## Infusion (Module) System

Modules live in `/infusions/<name>/`. Each has:

```
infusions/mymodule/
├── manifest.json       # Module metadata & SDK contract declarations
├── bootstrap.php       # Loaded on every page if module is enabled
├── admin.php           # Admin panel page (registered via manifest)
├── panel.php           # Frontend panel widget (registered via manifest)
├── schema.php          # Database schema (install/uninstall)
├── uninstall.php       # Uninstall logic
├── upgrade.php         # Version migration logic
├── classes/            # Module-specific OOP classes
├── migrations/         # Versioned SQL migrations
├── assets/             # CSS/JS files
├── support/            # Support files
└── locale/             # i18n strings
```

### manifest.json Structure

```json
{
  "name": "Module Name",
  "description": "...",
  "version": "1.0.0",
  "author": "...",
  "admin": true,
  "bootstrap": true,
  "panel": true,
  "schema": true,
  "upgrade": false,
  "module_class": "App\\MyModule\\MyModuleModule",
  "default_position": "right",
  "default_panel_name": "Panel Title",
  "min_core_version": "1.0.0",
  "min_php_version": "8.0.0",
  "required_extensions": ["json", "pdo", "pdo_mysql"],
  "dependencies": [],
  "conflicts": [],
  "permissions": [
    {"slug": "mymodule.view", "name": "View", "description": "..."}
  ],
  "provides": {
    "panels": ["mymodule"],
    "permissions": ["mymodule.view"],
```

## Security

### CSRF Protection

```php
// Generate token in form — two equivalent ways:
echo '<input type="hidden" name="csrf_token" value="' . generate_csrf_token() . '">';
echo csrf_field(); // shorthand helper

// Validate on form submit:
validate_csrf_token($_POST['csrf_token']);  // dies on failure
```

### XSS Prevention

```php
// Always escape output:
echo e($user_input);    // shorthand in helpers.php
echo esc($user_input);  // alias
echo htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8'); // explicit
```

### Logging

```php
// Audit log (admin actions)
log_audit_action($pdo, $user_id, 'user.deleted', 'user', $target_id, ['username' => $name]);

// Security log (auth events)
log_security_event($pdo, 'login.failed', ['email' => $email, 'ip' => $_SERVER['REMOTE_ADDR']]);
```

### Security Headers (set in bootstrap.php)

- **CSP**: Strict, allows only self + youtube-nocookie for iframes
- **HSTS**: Enabled (HTTPS)
- **X-Frame-Options**: SAMEORIGIN
- **X-Content-Type-Options**: nosniff
- **Referrer-Policy**: strict-origin-when-cross-origin

## Panel System

Panels are widgets rendered at positions on frontend pages.

```php
// Render panels at a position (in theme templates)
render_panels('left');       // Left sidebar
render_panels('u_center');   // Upper center
render_panels('l_center');   // Lower center
render_panels('right');      // Right sidebar
render_panels('au_center');  // All-upper center
render_panels('bl_center');  // Bottom-left center
```

Panel render lifecycle helpers in `includes/functions/output.php`: `panel_render_begin()`, `panel_render_state()`, `panel_render_mark_custom_shell()`.

Panels are managed via Admin → Panels (drag & drop reordering, enable/disable per panel).

## Module Settings Pattern

Modules store settings in `settings` using a `<module>_` prefix:

```php
// Read with default fallback
$value = setting('downloads_max_file_size', '52428800');

// Save
$stmt = $pdo->prepare("INSERT INTO settings (setting_key, setting_value)
    VALUES (?, ?) ON DUPLICATE KEY UPDATE setting_value = ?");
$stmt->execute([$key, $value, $value]);
```

## Localization

```php
// Translate a key
echo __('nav.home');

// With parameter substitution
echo __('member.reputation.breakdown', ['shoutbox' => 5, 'forum' => 12, 'comments' => 3]);

// Fallback: if key not found, returns the key itself
// Custom default: __('some.key', [], 'Default text')
```

### How it works

- Active locale read from `settings.site_locale` (default: `lt`)
- Core strings: `/locale/lt.php` — returns a flat `['key' => 'value']` array
- Module strings: `/infusions/<name>/locale/lt.php` — merged on top of core, same format
- All locale files are auto-loaded at bootstrap; module locales loaded only for enabled modules
- Locale keys use dot notation by convention: `nav.home`, `forum.topic.title`, etc.

### Adding strings

```php
// In locale/lt.php or infusions/<name>/locale/lt.php
return [
    'mymodule.label' => 'Mano modulis',
    'mymodule.msg'   => 'Sveiki, :name!',
];
```

## Development Conventions

1. **No ORM** — write raw SQL with PDO prepared statements
2. **Escape all output** — use `e()` or `htmlspecialchars()` before echoing user data
3. **CSRF on all state-changing forms** — use `csrf_field()` in form, `validate_csrf_token()` on POST
4. **Permissions before actions** — call `has_permission()` before any privileged operation
5. **Audit sensitive actions** — call `log_audit_action()` for admin operations
6. **New modules** — follow infusion structure, register permissions in manifest.json
7. **Schema changes** — add new numbered SQL file in `/database/`, update `database.sql`
8. **Admin pages** — always `require_once '_guard.php'` at the top
9. **Per-module admin** — route through `infusion-admin.php?folder=<name>`, not standalone files
